For loops

Voordat we met for loops aan de slag gaan, zullen we eerst de range behandelen.

range

range lijkt zeer veel op een array. Het is dan ook een verzameling. Maar waar we bij een array zelf konden bepalen wat we in de verzameling stoppen, is dit bij range al voor ons bepaald. range is een getallenreeks. Het bevat alleen integers, in volgorde. Het bereik van de range (welke getallen er in zitten) geven we aan wanneer we de range aanmaken.

We maken een range aan met de range(b,e,s) functie. Deze functie neemt 3 argumenten:
Positie Naam Optioneel ? Default
1e Begin Ja 0
2e Eind Nee NvT
3e Stap Ja 1

We kunnen dus een range aanmaken met 1, 2, of 3 argumenten.

Doen we dit met 1 arugment, dan hebben we alleen argument 2. Dan krijgen we dus een getallenreeks vanaf 0, TOT ons argument. dus niet tot en met.

Geven we 2 argumenten, dan hebben we argument 1 en 2. In dit geval krijgen we een getallenreeks vanaf argument 1, TOT argument 2.

Geven we alle 3 argumenten mee, dan krijgen we een range vanaf argument 1, TOT argument 2, met stapgrootte van argument 3. Stapgrootte houdt in dat we zoveel getallen verder gaan voor ieder volgend element in de reeks. Stapgrootte 2 slaat dus telkens 1 getal over. Stapgrootte 5 gaat 5 getallen verder ieder volgend element.

For loops

De for loop is tot op zekere hoogte te vergelijken met de while loop. Er zijn echter een paar verschillen. De while loop keek telkens of de conditie die erbij hoorde nog True was, en zo ja, voerde deze het code block nogmaals uit. De for loop heeft in plaats van een conditie, een verzameling en een iterator. De iterator is niets anders dan een variabele die bij de for loop hoort.

Wanneer we een for loop beginnen, zetten we de waarde van de iterator op het eerste element van de verzameling. We voeren dan het code block van de for loop uit. Daarna wordt de waarde van het volgende element aan de iterator gehangen. We voeren nogmaals het code block uit. Dit doen we voor (for) ieder element van de verzameling.

De for loop stopt dus wanneer het code block is uitgevoerd met de laatste waarde van de verzameling in de iterator. De conditie die de for loop dus eigenlijk iedere iteratie bekijkt, is "Is er nog een element over in de verzameling?".

Het maakt voor een for loop niet uit wat voor verzameling deze overheen loopt. Het kan iedere mogelijke verzameling zijn.
Verzameling Van
array Wat we er in stoppen
string karakters
range integers

Nested loops

Net als bij while loops, is het belangrijk om te zien hoe vaak een binnenste loop wordt uitgevoerd wanneer deze in het code block van een andere loop staat.
Positie Aantal keer uitvoeren (alleen) Aantal keer uitvoeren (nested)
Buitenste x x
Binnenste y x * y

Loopen over indexen

Omdat for loops al van zichzelf werken met een verzameling, zijn ze nog meer geschikt voor loopen over arrays dan while loops. Bij de while loop zagen we dat we met een integer begonnen, en die telkens als index gebruikten om elementen uit een verzameling op te vragen. We kunnen dit bij de for loop ook zo doen, met behulp van range, maar er is een handigere manier.

Omdat de for loop direct de waarde van de elementen van de verzameling in de iterator stopt, kunnen we de array waar we overheen willen loopen gebruiken als de verzameling van de for loop zelf.

De iterator heeft dan direct de waarden van de elementen van deze array, en daar kunnen we in het code block van de for loop meteen mee aan de slag.
