While loops

Tot nu toe hebben we onze programma's in verschillende situaties verschillende dingen laten doen. Maar onze programma's hebben dat tot nu toe maar één keer uitgevoerd. In een "echt" programma willen we vaak bepaalde acties meerdere keren uitvoeren. Denk bijvoorbeeld aan de website google, we willen ALLE resultaten weergeven, niet slechts één.

Wanneer we een bepaalde actie meerdere malen willen uitvoeren in onze programma's gebruiken we iets genaamd een loop. De eerste constructie voor een loop die we zullen behandelen, is de while loop.

De while loop heeft veel overeenkomsten met de if. Beiden beginnen met een keyword, gevolgd door een conditie, en dan een code block. Het verschil tussen de while en de if is dat de if zijn code block een keer uitvoert als (if) de conditie True is. De while loop blijft echter zijn code block opnieuw en opnieuw uitvoeren zolang (while) de conditie True is.

Wanneer ons programma aankomt bij een while keyword, kijken we eerst of de conditie True is. Als dit zo is, voeren we het code block dat bij de while hoort uit. Daarna kijken we nogmaals of de conditie nog steeds True is, en blijven we dit proces doorgaan tot de conditie False is.

Als we dus willen dat de while loop ooit ook eens stopt, moeten we er voor zorgen dat de conditie ooit False wordt. Doen we dit niet, dan zal de loop voor altijd blijven doorgaan, en zal ons programma nooit voorbij de loop kunnen komen. Omdat we in de loop alleen het bijbehorende code block uitvoeren, moeten we dus in dit code block zorgen dat de conditie weer False wordt.

Nested loops

Wanneer we een loop in het code block van een andere loop plaatsen (nesten), is het belangrijk om in te zien hoe vaak de binnenste loop uitgevoerd zal worden.
Positie Aantal keer uitvoeren (alleen) Aantal keer uitvoeren (nested)
Buitenste x x
Binnenste y x * y

Bij while loops zal dit alleen het geval zijn als we in de buitenste loop de conditie van de binnenste loop "resetten". Bij andere loop constructies is dit niet nodig, en zal het bovenstaande schema altijd van toepassing zijn.

Loopen over indexen

Tot nu toe hebben we in onze loops telkens een integer verhoogd of verlaagd. We kunnen deze integer natuurlijk ook als index van een array gebruiken. Wat er dan gebeurt, is dat in plaats van de getallen 0,1,2 etc. , pakken we het 1e, 2e, 3e element van een array.

Wanneer we loopen over indexen, is het belangrijk te onthouden dat indexen beginnen bij 0, en eindigen bij len(array) -1. De hoogste index is dus altijd kleiner dan de lengte van de array. Dit houdt in dat als we een variabele gebruiken om de index aan te geven, dat we moeten zorgen dat die variabele altijd kleiner blijft dan de lengte van de array.

Als we bijvoorbeeld een array met lengte 4 zouden hebben, en we proberen daar index 4 uit te pakken, zullen we een error krijgen. De hoogste index in deze array is namelijk
(len(array) - 1) == (4 - 1) == 3
